# English

## Overview

Ttobak is an AI-powered meeting assistant built for AWS Solutions Architects. Record meetings, get real-time transcription and translation, generate AI summaries, and access customer intelligence through automated news crawling and deep research.

**Live**: [ttobak.atomai.click](https://ttobak.atomai.click)

## Features

### Recording and Transcription
- **Microphone recording** with real-time captions (Web Speech API / AWS Transcribe Streaming)
- **Tab Audio capture** for Google Meet via Chrome `getDisplayMedia` API
- **Audio file upload** (m4a, mp3, wav, webm) for post-meeting transcription
- **A/B STT comparison** with AWS Transcribe and Nova Sonic
- **Real-time translation** during recording (6 languages via Amazon Translate)

### AI Analysis
- **Meeting summary** generated by Bedrock Claude Sonnet 4.6
- **Action item extraction** with assignees and priorities
- **Auto-tagging** of meeting topics
- **Q&A with RAG** — ask questions about meetings using Bedrock Knowledge Base

### Customer Intelligence (Insights)
- **News crawling** — automated Google News RSS collection for subscribed customers
- **Tech docs crawling** — AWS documentation per customer's tech stack
- **AI briefings** — Sonnet 4.6 generates SA-focused analysis for each article
- **Deep Research** — Bedrock Agent performs multi-source research with citation-backed reports
- **Shared KB** — crawled content is searchable via QA RAG

### Integrations
- **MCP Server** — Claude Code integration via OAuth PKCE for meeting data access
- **Notion export** — one-click export of meeting notes
- **Obsidian/PDF export**

### Infrastructure
- **CI/CD** — GitHub Actions with self-hosted runner on EC2
- **9 CDK stacks** — fully automated infrastructure deployment
- **CloudFront + Lambda@Edge** — JWT auth at edge, S3 OAC for static content

## Architecture

```
CloudFront (ttobak.atomai.click)
  +-- Lambda@Edge (JWT auth on /api/*)
  +-- S3 OAC (Next.js static export)
  +-- HTTP API Gateway --> ttobak-api Lambda (Go, chi router)

Event-Driven Pipeline:
  audio/ upload --> EventBridge --> ttobak-transcribe --> AWS Transcribe
  transcripts/ --> EventBridge --> ttobak-summarize --> Bedrock Claude --> DynamoDB
  images/ --> EventBridge --> ttobak-process-image --> Bedrock Vision

Crawler Pipeline:
  EventBridge (daily) --> Step Functions --> TechCrawler + NewsCrawler --> S3 KB --> Bedrock KB

Research Pipeline:
  POST /api/research --> Bedrock Agent (Deep Research) --> S3 KB

QA Pipeline:
  POST /api/qa/ask --> Bedrock Converse (Sonnet 4.6) + Tools (KB, AWS Docs, Transcript)
```

### CDK Stack Dependency

```
Auth + Storage (parallel)
  --> AI --> Knowledge --> EdgeAuth (us-east-1) --> Gateway --> Crawler --> ResearchAgent --> Frontend
```

| Stack | Purpose |
|-------|---------|
| AuthStack | Cognito User Pool, SPA Client, Identity Pool |
| StorageStack | DynamoDB (single-table), S3 assets bucket |
| AiStack | IAM roles for all Lambda functions |
| KnowledgeStack | OpenSearch Serverless, Bedrock KB, KB S3 bucket |
| EdgeAuthStack | Lambda@Edge JWT validation (us-east-1) |
| GatewayStack | API Gateway, 8 Lambda functions, EventBridge rules, WebSocket API |
| CrawlerStack | Step Functions, 4 crawler Lambdas, daily EventBridge schedule |
| ResearchAgentStack | Bedrock Agent for deep research, tool Lambdas |
| FrontendStack | S3 site bucket, CloudFront distribution, SPA router |

## Getting Started

### Prerequisites

- Go 1.24+
- Node.js 20+
- Python 3.12+
- AWS CLI (configured)
- AWS CDK CLI (`npm install -g aws-cdk`)

### Local Development

```bash
# Frontend dev server
cd frontend && npm install && npm run dev

# Backend build (ARM64 Lambda binaries)
cd backend
for dir in cmd/api cmd/transcribe cmd/summarize cmd/process-image cmd/kb; do
  GOOS=linux GOARCH=arm64 /usr/local/go/bin/go build -tags lambda.norpc -o $dir/bootstrap ./$dir
done
```

### Deploy

```bash
# Infrastructure (all 9 stacks)
cd infra && npm install && npx cdk deploy --all

# Frontend
cd frontend && npm run build
aws s3 sync out/ s3://ttobak-site-{ACCOUNT_ID}-ap-northeast-2/ --delete
aws cloudfront create-invalidation --distribution-id {DIST_ID} --paths "/*"
```

### CI/CD

Push to `main` triggers automatic deployment via GitHub Actions. Manual trigger available via `workflow_dispatch`.

# 한국어

## 개요

Ttobak(또박)은 AWS Solutions Architect를 위한 AI 미팅 어시스턴트입니다. 미팅을 녹음하고, 실시간 전사 및 번역을 제공하며, AI 요약을 생성하고, 자동화된 뉴스 크롤링과 딥 리서치를 통해 고객 인텔리전스에 접근할 수 있습니다.

**Live**: [ttobak.atomai.click](https://ttobak.atomai.click)

## 주요 기능

### 녹음 및 전사
- **마이크 녹음** — 실시간 자막 지원 (Web Speech API / AWS Transcribe Streaming)
- **탭 오디오 캡처** — Chrome `getDisplayMedia`로 Google Meet 오디오 녹음
- **오디오 파일 업로드** — m4a, mp3, wav, webm 파일로 후처리 전사
- **A/B STT 비교** — AWS Transcribe와 Nova Sonic 동시 변환, 결과 비교
- **실시간 번역** — 녹음 중 Amazon Translate로 6개 언어 번역

### AI 분석
- **미팅 요약** — Bedrock Claude Sonnet 4.6으로 자동 요약 생성
- **액션 아이템 추출** — 담당자, 우선순위 포함
- **자동 태깅** — 미팅 주제 자동 분류
- **RAG Q&A** — Bedrock Knowledge Base로 미팅 내용 질문/답변

### 고객 인텔리전스 (Insights)
- **뉴스 크롤링** — 구독한 고객사의 Google News RSS 자동 수집
- **기술 문서 크롤링** — 고객사 기술 스택별 AWS 문서 수집
- **AI 브리핑** — Sonnet 4.6이 각 기사에 대한 SA 관점 분석 생성
- **딥 리서치** — Bedrock Agent가 다출처 웹 리서치 수행, 인용 기반 리포트 생성
- **공유 KB** — 크롤링된 콘텐츠가 QA RAG로 검색 가능

### 연동
- **MCP 서버** — OAuth PKCE 인증으로 Claude Code에서 미팅 데이터 접근
- **Notion 내보내기** — 원클릭 미팅 노트 내보내기
- **Obsidian/PDF 내보내기**

### 인프라
- **CI/CD** — GitHub Actions + EC2 self-hosted runner 자동 배포
- **9개 CDK 스택** — 완전 자동화된 인프라 배포
- **CloudFront + Lambda@Edge** — 엣지에서 JWT 인증, S3 OAC로 정적 콘텐츠 제공
